# UI Network Discovery Update

## Purpose

The UI now shows how counterparties enter the system before invoice financing begins. This closes the earlier product gap where it was unclear how buyers find sellers, sellers find buyers, and financiers discover deals.

## New UI modules

### Business Directory

Shows discoverable verified profiles for:

- buyers
- sellers
- financiers
- logistics providers

Users can inspect capabilities, score and role, then invite/connect.

### Counterparty Network

Shows private invites and accepted relationships.

Supports:

- known-counterparty invite
- directory invite
- financier relationship
- logistics partnership

### Trade Contracts

Shows buyer-led or seller-led contracts.

Supports:

- create contract
- invite seller
- accept/request changes
- activate into order

### Trade Opportunities

Shows buyer-posted demand that sellers can discover.

Supports:

- buyer posts opportunity
- seller submits proposal
- buyer reviews proposal

### Seller Proposals

Shows proposals submitted by sellers.

Supports:

- submitted
- shortlisted
- accepted
- rejected
- contract conversion

### Financier Marketplace

Shows finance-ready receivables discoverable by financiers.

Supports:

- deal discovery
- recommended advance
- proof tx visibility
- express interest

## Product framing

The UI should frame Credara as:

> A verified trade finance network where buyers and sellers create trusted trade contracts, sellers convert verified invoices into finance-ready receivables, and financiers discover fundable deals backed by proof, KYB, Smart LC settlement and Polygon receipts.

## Relationship models visible in UI

1. Private invite
2. Verified directory
3. Buyer opportunity and seller proposal
4. Financier marketplace
5. Embedded/API relationship creation
